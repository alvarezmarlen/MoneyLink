# USER HISTORY

## ID
US-006

## Título
Seguridad y resguardo de datos personales

## COMO:
Usuario que envía remesas internacionales

## QUIERO:
Tener certeza de que mis datos personales y bancarios están protegidos

## PARA:
Garantizar la seguridad de mi información sensible

---
## Criterios de Aceptación

## Escenarios

Escenario 1 — Almacenamiento seguro de datos  
DADO: que el usuario introduce datos personales y bancarios en la aplicación  
CUANDO: la información se guarda en el sistema  
ENTONCES: los datos deben almacenarse en un backend local protegido y nunca mostrarse públicamente  

---

Escenario 2 — Acceso restringido a la información  
DADO: que el usuario tiene datos guardados en el sistema  
CUANDO: accede a la sección de gestión de datos  
ENTONCES: solo el propio usuario puede ver, editar o eliminar su información  

---

Escenario 3 — No exposición de datos sensibles  
DADO: que el usuario navega por la aplicación  
CUANDO: se muestran datos en pantalla  
ENTONCES: nunca deben mostrarse números completos de cuenta, documentos o información sensible sin enmascarar  

---

Escenario 4 — Eliminación segura  
DADO: que el usuario decide borrar sus datos personales  
CUANDO: confirma la eliminación  
ENTONCES: el sistema debe eliminar permanentemente la información del backend local  

---

Escenario 5 — Manejo de errores  
DADO: que ocurre un fallo al guardar o cargar datos personales  
CUANDO: el sistema detecta el error  
ENTONCES: se debe mostrar un mensaje claro sin revelar información sensible  

---

## Notas
- Los datos sensibles deben enmascararse visualmente.  
- JSON Server no cifra datos, por lo que se deben evitar campos innecesarios.  
- No almacenar contraseñas ni información no requerida.  
- Cumplir buenas prácticas de privacidad.
