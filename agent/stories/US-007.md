# USER HISTORY

## ID
US-007

## Título
Seguimiento del envío en tiempo real

## COMO:
Usuario que envía remesas internacionales

## QUIERO:
Tener información de la ruta de mi envío desde origen hasta destino

## PARA:
Hacer seguimiento en tiempo real y conocer el estado del proceso

---
## Criterios de Aceptación

## Escenarios

Escenario 1 — Visualización del estado del envío  
DADO: que el usuario ha realizado un envío  
CUANDO: accede a la sección de seguimiento  
ENTONCES: la aplicación debe mostrar el estado actual del envío (procesando, en tránsito, recibido)

---

Escenario 2 — Línea de tiempo del envío  
DADO: que el envío tiene varias etapas  
CUANDO: el usuario abre el detalle del seguimiento  
ENTONCES: se muestra una línea de tiempo con cada etapa y su fecha/hora

---

Escenario 3 — Actualización en tiempo real  
DADO: que el envío está en curso  
CUANDO: el estado cambia en el backend o API  
ENTONCES: la aplicación debe actualizar la información sin necesidad de recargar manualmente

---

Escenario 4 — Información de ubicación  
DADO: que el envío está en tránsito  
CUANDO: el usuario consulta la ruta  
ENTONCES: se debe mostrar la ubicación aproximada o etapa actual del proceso

---

Escenario 5 — Notificación de entrega  
DADO: que el envío ha sido entregado  
CUANDO: el sistema recibe la confirmación  
ENTONCES: el usuario debe ver claramente que el envío ha llegado a destino

---

Escenario 6 — Manejo de errores  
DADO: que ocurre un fallo al obtener el estado del envío  
CUANDO: la aplicación intenta cargar la información  
ENTONCES: se muestra un mensaje claro y se ofrece reintentar

---

## Notas
- El seguimiento puede simularse con JSON Server en este sprint.  
- La actualización en tiempo real puede implementarse con polling simple.  
- La línea de tiempo puede ser un componente reutilizable.
