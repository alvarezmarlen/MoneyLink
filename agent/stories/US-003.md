# Historia de Usuario

## ID
US-003

## Título
Proceso de envío de divisas y selección de método de pago

## Descripción
**Como** usuario remitente,
**Quiero** ingresar los datos del destinatario con opción a guardarlo como frecuente, llenar mis propios datos, seleccionar el método de pago y ver un resumen detallado de la transacción
**Para** efectuar el envío de divisas de forma clara, tener visibilidad del tipo de cambio y los costos, y completar el pago de manera segura.

## Criterios de Aceptación

### Escenario 1: Ingreso de datos del destinatario y tener la opción de marcado como frecuente
```gherkin
Funcionalidad: Ingreso de datos de quien recibe las divisas
  Escenario: Validar datos y pasar la informacion del remitente
    Dado que el usuario se encuentra en la vista de datos del destinatario
    Cuando ingresa los datos de la persona que recibe el dinero
    Y tiene la opción de seleccionarla como "usuario frecuente"
    Entonces el sistema valida la información ingresada
    Y muestra la siguiente vista para rellenar la información del usuario que envía
```

### Escenario 2: Selección de método de pago y visualización de resumen final
```gherkin
Funcionalidad: Métodos de pago y resumen
  Escenario: Seleccionar método de pago y mostrar resumen de costos
    Dado que el usuario ha llenado los datos con su información de remitente
    Cuando el sistema muestra el apartado de métodos de envío
    Y el usuario selecciona si efectuar el envío por transferencia de banco, tarjeta de débito o PayPal
    Entonces aparece de último el resumen de lo que se envía, incluyendo el exchange del monto y lo que se cobrará por enviar el dinero
    Y se muestra un botón de "Continuar y pagar"
```

## Notas
- Se debe considerar almacenar los usuarios frecuentes en el jsonserver vinculados al perfil del remitente.
- El resumen debe calcular con Exchange Rate API (tipo de cambio) actualizado al momento de la operación y desglosar de forma transparente la comisión.
- Sería ideal manejar los errores si la validación del destinatario falla.
