# Historia de Usuario

## ID
US-004

## Título
Visualización del dashboard de transacciones

## Descripción
**Como** usuario de la plataforma
**Quiero** visualizar un dashboard con la información de mi transacción en curso, el historial de transacciones y mis destinatarios frecuentes, además de un botón para iniciar una nueva transacción
**Para** gestionar fácilmente mis operaciones y realizar nuevas transacciones de forma rápida.

## Criterios de Aceptación

### Escenario 1: Visualizar transacción en curso
```gherkin
Dado que el usuario ha iniciado sesión
Cuando accede al dashboard
Entonces debe visualizar la información de la transacción en curso si existe
```

### Escenario 2: Visualizar historial de transacciones
```gherkin
Dado que el usuario ha iniciado sesión
Cuando accede al dashboard
Entonces debe visualizar el listado de sus transacciones anteriores
```

### Escenario 3: Visualizar destinatarios frecuentes
```gherkin
Dado que el usuario ha iniciado sesión
Cuando accede al dashboard
Entonces debe visualizar la lista de sus destinatarios frecuentes
Y debe poder seleccionar uno para iniciar una nueva transacción hacia ese destinatario
```

### Escenario 4: Iniciar una nueva transacción desde el botón principal
```gherkin
Dado que el usuario se encuentra en el dashboard
Cuando hace clic en el botón "Nueva transacción"
Entonces debe ser redirigido a la página de inicio del flujo de transacción
```

## Notas
- La información debe cargarse de forma segura y en tiempo real.
- Si no existe transacción en curso, la sección no debe mostrarse o debe indicar que no hay procesos activos.
- El listado de destinatarios frecuentes debe mostrar al menos nombre y referencia básica.
- El diseño debe priorizar la usabilidad y rapidez de acceso a una nueva transacción.
