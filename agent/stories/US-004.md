## Título Ingreso inicial y autollenado básico de datos de remitente y destinatario

COMO:     Usuario que envía remesas internacionales
QUIERO: Mantener historial de datos básicos de remitente y destinatario
PARA: Facilitar/agilizar futuros trámites de envíos de divisas

```gherkin

  Escenario 1: Guardado automático de datos del remitente
  Given el usuario completa el formulario de envío por primera vez con sus datos de remitente (nombre, identificación, número de cuenta, dirección, teléfono y país)
  When confirma el envío o guarda la información
  Then el sistema almacena estos datos en el backend local para reutilizarlos en futuros envíos

  Escenario 2: Guardado automático de datos del destinatario
  Given el usuario introduce los datos del destinatario en el formulario (nombre, identificación, número de cuenta, dirección, teléfono y país)
  When confirma el envío o guarda la información
  Then el sistema guarda estos datos en el backend local como destinatario frecuente

  Escenario 3: Auto llenado del remitente en futuros envíos
  Given el usuario ya tiene datos de remitente guardados
  When accede nuevamente al formulario de envío
  Then los campos del remitente aparecen autocompletados automáticamente, permitiendo editar cualquier dato antes de enviar

  Escenario 4: Selección de destinatario frecuente
  Given el usuario tiene varios destinatarios guardados
  When abre el formulario de envío
  Then puede seleccionar un destinatario desde una lista y los datos se autocompletan automáticamente en el formulario

  
```
