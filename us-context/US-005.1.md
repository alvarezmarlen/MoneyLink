```gherkin
Escenario 1: Edición de datos guardados
DADO: que el usuario accede a la sección de gestión de perfiles
CUANDO: modifica los datos de un remitente o destinatario y guarda los cambios
ENTONCES: el sistema actualiza la información almacenada en el backend local

DADO: que el usuario tiene destinatarios guardados
CUANDO: selecciona la opción "Eliminar" en uno de ellos
ENTONCES: el sistema solicita confirmación y, si el usuario acepta, elimina el destinatario del backend local

Escenario 3: Persistencia de datos 
DADO: que el usuario ha guardado datos previamente
CUANDO: vuelve a abrir la aplicación
ENTONCES: los datos de remitente y destinatarios se cargan automáticamente desde el backend local sin necesidad de volver a escribirlos

Escenario 4: Selección de destinatario frecuente
DADO: que el sistema intenta cargar los datos guardados
CUANDO: ocurre un error en la lectura del backend local
ENTONCES: se muestra un mensaje claro de error y se ofrece la opción de reintentar o continuar sin autollenado
```
