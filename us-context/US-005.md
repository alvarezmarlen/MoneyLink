# Historia de Usuario

## ID
US-005

## Título
Gestión avanzada de perfiles y destinatarios frecuentes (edición, eliminación y persistencia)

## Descripción
**Como** usuario autenticado en la aplicación
**Quiero** editar, eliminar y mantener persistentes mis datos personales y los de mis destinatarios frecuentes
**Para** gestionar correctamente mi información y agilizar futuras transferencias internacionales

## Criterios de Aceptación

### Escenario 1: Editar información del perfil del usuario
```gherkin
Dado que el usuario ha iniciado sesión
Y accede a la sección de perfil
Cuando modifica su información personal y guarda los cambios
Entonces el sistema debe actualizar los datos en el backend local
Y mostrar un mensaje claro de confirmación
```

### Escenario 2: Validación de datos del perfil
```gherkin
Dado que el usuario está editando su perfil
Cuando ingresa información inválida o incompleta
Entonces el sistema debe mostrar mensajes de validación
Y no permitir guardar los cambios hasta corregir los errores
```

### Escenario 3: Editar información de un usuario frecuente
```gherkin
Dado que el usuario tiene destinatarios guardados
Y accede a la lista de destinatarios frecuentes
Cuando selecciona uno y edita su información
Entonces el sistema debe guardar los cambios correctamente
Y reflejar la información actualizada en la lista
```

### Escenario 4: Eliminación de un destinatario frecuente
```gherkin
Dado que el usuario visualiza la lista de destinatarios frecuentes
Cuando selecciona la opción "Eliminar" en uno de ellos
Entonces el sistema debe solicitar confirmación
Y si el usuario acepta, eliminar el destinatario del backend local
Y actualizar la lista en pantalla
```
### Escenario 5: Persistencia automática
```gherkin
Dado que el usuario ha guardado datos previamente
Cuando vuelve a abrir la aplicación
Entonces el sistema debe cargar automáticamente el perfil y los destinatarios desde el backend local
Sin necesidad de volver a escribirlos
```
### escenario 6: Cancelación de edición
```gherkin
Dado que el usuario está editando su perfil o un destinatario frecuente
Cuando selecciona la opción "Cancelar"
Entonces el sistema no debe guardar los cambios
Y debe mostrar la información original
```

## Notas
- Requiere CRUD completo en backend local (JSON Server o equivalente).

- Debe incluir validaciones de formato, campos obligatorios y mensajes de error claros.

- Debe incluir mensajes de confirmación para edición y eliminación.

- La persistencia debe ser automática y transparente para el usuario.

- La interfaz debe seguir el estilo MoneyLink (Glassmorphism + responsive).
